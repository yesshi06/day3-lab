## 1. 背景与问题

> 投研人员在对非上市公司（如一级市场目标公司）进行估值时，面临以下痛点：

| 痛点 | 来源 |
|------|------|
| 估值流程手工化：需手动查找可比公司、手动录入财务数据、手动计算PE/DCF等指标，效率低下 | 用户访谈 |
| 可比公司选择不稳定：不同分析师对同一目标公司选取的可比公司不一致，缺乏系统化推荐与确认机制 | 团队讨论 |
| 多种估值方法难以并行对比：PE可比法、DCF现金流折现法等多种方法的参数和结果分散在不同Excel中，缺乏统一视图 | 用户访谈 |
| 财务数据获取繁琐：上市公司三表（资产负债表、利润表、现金流量表）及历史股价数据需要跨多个系统手动抓取 | 邮件沟通 |

## 2. 提案概要

构建一套**公司估值模型系统**，帮助投研人员完成从目标公司选定、可比公司AI推荐与确认、财务数据导入、多方法估值计算到结果综合展示的全链路估值流程。系统支持 PE 可比法、DCF 现金流折现法等多种估值方法并行计算，输出估值区间与推荐方法，供业务部门直接使用。后端采用 Flask + JSON 文件存储，前端采用 React + Vite，测试使用 pytest。

## 3. 范围边界

| 做（范围内） | 不做（范围外） |
|-------------|---------------|
| 目标公司选择/搜索/筛选 | 实时行情推送与交易功能 |
| 目标公司详情：财务三表导入、行业情况展示 | 多租户/行级数据权限管理 |
| 可比公司 AI 推荐（4~6家）+ 用户确认/调整 | 自动化投资决策与下单 |
| 可比公司财务数据与历史股价展示 | 期权/衍生品估值 |
| 多种估值方法并行计算（PE可比法、DCF、P/B、EV/EBITDA） | 实时数据爬虫（教学版用手动导入/模拟数据） |
| 估值结果综合展示：各方法估值区间 + AI 推荐总结 | 审计合规报告生成 |

## 4. 成功标准（可度量）

| 标准 | 度量方式 |
|------|----------|
| 主路径可走通 | 用户在 5 分钟内完成"选定目标公司 → 确认可比公司 → 查看估值结果"全链路 |
| 估值方法覆盖 | 系统至少支持 3 种估值方法（PE可比法、DCF、P/B）并行输出结果 |
| 可比公司推荐有效 | AI 推荐的可比公司中，用户确认采用率 ≥ 60% |
| 估值结果可用 | 各方法估值结果均附带参数明细与计算过程，可供人工校验 |

## 5. 风险（简表）

| 风险 | 影响 | 应对 |
|------|------|------|
| AI 推荐可比公司不稳定 | 每次推荐结果不一致，用户信任度低 | 采用"AI推荐 + 用户确认"机制，推荐结果可人工调整 |
| 财务数据获取受限 | 教学环境无法实时抓取上市公司数据 | 教学版使用模拟数据/手动导入；生产版接入数据API |
| 估值模型参数敏感 | DCF等方法参数假设不同导致结果差异大 | 展示参数明细，支持用户调参重算 |

## 6. 立项前置条件（Go / No-Go）

- ☐ 后端与前端可独立启动
- ☐ 至少准备 3 家公司的模拟财务数据（三表 + 股价）
- ☐ 估值引擎（PE计算）核心算法可运行
- ☐ AI 可比公司推荐接口可用（或 Demo 模式兜底）
