## 2. 技术选型 ADR

| 层 | 选型 | 理由 | 替代方案 |
|----|------|------|----------|
| 前端框架 | React 18 + Vite | 团队已有经验，热重载开发效率高 | Vue 3（团队不熟悉） |
| 样式 | TailwindCSS | 原子化 CSS，快速构建金融风格 UI | Ant Design（过重） |
| 后端框架 | Flask 3.x | 轻量灵活，适合 API 服务 | FastAPI（异步非必需） |
| 存储 | JSON 文件 | 教学版零依赖，快速启动 | SQLite（演进方案） |
| AI 推荐 | LLM API（百炼/CoPaw） | 可比公司推荐需要行业理解 | 规则引擎（不够智能） |
| 估值引擎 | Python 原生计算 | PE/DCF/P/B 算法简单明确 | NumPy（当前无需） |
| 测试 | pytest | Python 生态标准 | unittest（API 不够简洁） |

## 3. 估值引擎设计

### 3.1 PE 可比法

```
输入：目标公司净利润、可比公司列表(含股价+EPS)
计算：
  1. 各可比公司 PE = 股价 / EPS
  2. PE 中位数 = median(所有可比公司 PE)
  3. 估值 = PE 中位数 × 目标公司净利润
  4. 区间：[PE_min × 净利润, PE_max × 净利润]
输出：valuation_low / valuation_mid / valuation_high
```

### 3.2 DCF 现金流折现法

```
输入：基期现金流、折现率(默认10%)、增长率(默认5%)、预测年数(默认5)
计算：
  1. 各年预测现金流 = 基期 × (1+增长率)^n
  2. 各年折现值 = 预测现金流 / (1+折现率)^n
  3. 终值 = 末年现金流 × (1+永续增长率) / (折现率-永续增长率)
  4. 估值 = Σ折现值 + 终值折现
输出：valuation_low(±20%) / valuation_mid / valuation_high(±20%)
```

### 3.3 P/B 市净率法

```
输入：目标公司净资产、可比公司列表(含股价+每股净资产)
计算：
  1. 各可比公司 P/B = 股价 / 每股净资产
  2. P/B 中位数 = median(所有可比公司 P/B)
  3. 估值 = P/B 中位数 × 目标公司净资产
输出：valuation_low / valuation_mid / valuation_high
```

## 4. AI 推荐降级策略

```
Level 1: LLM API → 返回 4~6 家推荐公司 + 推荐理由
   ↓ 失败
Level 2: 行业规则匹配 → 同行业+相近营收规模的上市公司
   ↓ 失败
Level 3: 手动模式 → 提示用户手动输入可比公司
```

## 5. 存储演进路径

| 阶段 | 方案 | 适用场景 |
|------|------|----------|
| 教学版（当前） | JSON 文件 + RMW 模式 | 本地演示，≤100 家公司 |
| 演进一 | SQLite | 单机部署，≤10000 家 |
| 生产版 | PostgreSQL + Redis | 多人协作，外部数据源对接 |
