> **本文只做一件事**：按用户能理解的子场景拆成 5 条 **User Story**，并为每条写清 **REQ-ID**、**验收标准（AC）**。

## 字段说明 — 每条 UserStory 必须包含以下字段

| 字段 | 填写说明 | 数据来源 ← | 流向下游 → |
|------|----------|------------|-----------|
| **需求编号** | 唯一编号，如 REQ-M2VAL-001 | ← `02` 采集编号 | → `14` 追踪矩阵首列 |
| **故事** | 作为【角色】，我希望【行为】，以便【价值】 | ← `04` PRD 场景拆解 | → `06` FSD 功能点 |
| **优先级** | P0（必须）/ P1（应该）/ P2（可选） | ← `03` §4 成功标准 | → `12` 里程碑排期 |
| **来源** | 对应 03/04 哪一条需求 | ← `03` 范围 · `04` 场景 | |
| **前置条件** | 触发该故事的前提状态 | ← `04` 用户流程 | → `13` 测试前置 |
| **功能对齐** | 对应功能规格 + 非功能约束 | | → `06` FSD · `07` NFR |
| **设计对齐** | 对应架构、接口、数据、安全 | | → `08` 架构 · `09` API · `10` Data · `11` 安全 |
| **验证对齐** | 对应测试用例 + 追踪矩阵行 | | → `13` TC · `14` 追踪矩阵行 |
| **验收标准** | AC，每条可独立验证 | ← 故事 + `07` NFR | → `13` 测试用例依据 |

## US-001（P0）目标公司选择与筛选

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2VAL-001 |
| **故事** | 作为投研人员，我希望通过公司名称或行业指标筛选并选定一家目标公司，以便开始对该公司的估值分析流程 |
| **优先级** | **P0** |
| **来源** | ← 03 §3 范围边界「目标公司选择/搜索/筛选」 |
| **前置条件** | 系统已启动，公司数据库已有基础数据 |
| **功能对齐** | → 06 §公司筛选页 · 07 §响应时间 |
| **设计对齐** | → 08 §Flask · 09 GET /companies API · 10 companies.json 表 · 11 §数据访问 |
| **验证对齐** | → 13 TC-001 · 14 第1行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-001-01 | 公司列表页支持按公司名称关键字搜索，输入后 1s 内返回匹配结果 |
| AC-001-02 | 公司列表页支持按行业分类筛选 |
| AC-001-03 | 点击某家公司后，系统将其设为当前目标公司并跳转至公司详情页 |
| AC-001-04 | 已有估值记录的公司在列表中有标识，可快速回看 |

## US-003（P0）可比公司 AI 推荐与确认

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2VAL-003 |
| **故事** | 作为投研人员，我希望系统基于目标公司自动推荐 4~6 家可比上市公司，并由我确认/调整最终可比公司名单，以便获得稳定可控的估值样本 |
| **优先级** | **P0** |
| **来源** | ← 03 §3 范围边界「可比公司 AI 推荐 + 用户确认/调整」 |
| **前置条件** | 目标公司已选定且有基本信息（US-002 完成） |
| **功能对齐** | → 06 §可比公司推荐页 · 07 §推荐准确率 |
| **设计对齐** | → 08 §AI推荐引擎 · 09 POST /companies/{id}/comparables/recommend + PUT /companies/{id}/comparables · 10 comparables.json 表 · 11 §AI输出校验 |
| **验证对齐** | → 13 TC-003 · 14 第3行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-003-01 | 点击"AI推荐可比公司"后，系统返回 4~6 家推荐公司，附推荐理由（行业相似度、规模匹配度） |
| AC-003-02 | 用户可勾选/取消勾选推荐公司，也可手动添加公司到可比列表 |
| AC-003-03 | 用户确认后，可比公司名单锁定，后续估值计算基于该名单 |
| AC-003-04 | AI 不可用时，系统提示"推荐服务暂不可用"，用户可手动输入可比公司 |

## US-004（P0）多方法估值分析

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2VAL-004 |
| **故事** | 作为投研人员，我希望系统基于已确认的可比公司，自动运行多种估值方法（PE可比法、DCF、P/B等），并展示每种方法的计算过程和参数明细，以便对比不同方法的估值结果 |
| **优先级** | **P0** |
| **来源** | ← 03 §3 范围边界「多种估值方法并行计算」 |
| **前置条件** | 可比公司名单已确认（US-003 完成），可比公司财务数据与股价已就绪 |
| **功能对齐** | → 06 §估值分析页 · 07 §计算准确性 |
| **设计对齐** | → 08 §估值引擎 · 09 POST /companies/{id}/valuations/run + GET /companies/{id}/valuations/methods · 10 valuations.json 表 · 11 §计算结果完整性 |
| **验证对齐** | → 13 TC-004 · 14 第4行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-004-01 | 系统展示每家可比公司的核心财务指标（营收、净利润、股价、PE、P/B） |
| AC-004-02 | PE 可比法：取可比公司 PE 中位数 × 目标公司净利润 = 估值，计算过程全展示 |
| AC-004-03 | DCF 法：展示现金流预测、增速假设、折现率参数，用户可调参后重算 |
| AC-004-04 | P/B 法：取可比公司 P/B 中位数 × 目标公司净资产 = 估值 |
| AC-004-05 | 至少 3 种估值方法并行运行，每种方法单独展示计算详情卡片 |

## US-005（P0）估值结果综合展示

| 字段 | 内容 |
|------|------|
| **需求编号** | REQ-M2VAL-005 |
| **故事** | 作为投研人员，我希望在一个简洁的结果页查看所有估值方法的结果区间、估值中枢，以及 AI 对推荐方法的总结建议，以便快速形成估值判断 |
| **优先级** | **P0** |
| **来源** | ← 03 §3 范围边界「估值结果综合展示」 |
| **前置条件** | 至少一种估值方法已计算完成（US-004 完成） |
| **功能对齐** | → 06 §估值结果页 · 07 §展示完整性 |
| **设计对齐** | → 08 §结果聚合 · 09 GET /companies/{id}/valuations/summary · 10 valuations.json 表 · 11 §结果展示安全 |
| **验证对齐** | → 13 TC-005 · 14 第5行 |

| 验收标准 | 描述（每条可独立测试） |
|----------|----------------------|
| AC-005-01 | 结果页以卡片/表格形式展示各方法名称、估值区间（低-中-高）、核心参数 |
| AC-005-02 | 页面展示估值中枢（各方法中位数的加权/简单平均） |
| AC-005-03 | AI 总结区域：基于行业特征和可比公司质量，推荐最适合的估值方法及理由 |
| AC-005-04 | 支持导出估值报告摘要（JSON/文本格式） |
| AC-005-05 | 可从结果页一键返回调整可比公司或参数，重新计算 |
